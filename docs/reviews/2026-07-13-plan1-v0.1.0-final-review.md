## Fixes And Re-Verification

Two production fixes were required by the expanded Codex review:

1. `frontend/src/api/chat.test.ts::cancels the response reader after a terminal
   SSE error` reproduced zero calls to the stream's cancellation callback. The
   reader lifecycle is now guarded by `finally`, cancellation errors cannot
   hide the original stream error, and the lock is released.
2. `frontend/src/stores/chatStore.test.ts::ignores a stale conversation refresh
   response` reproduced the stale title replacing the newest title. Refreshes
   now carry an internal monotonic request identity and only the latest result
   may update state.

RED command:

```text
npm run test -- src/api/chat.test.ts src/stores/chatStore.test.ts
```

RED result: 2 files ran, 2 tests failed and 20 passed for the expected missing
reader cancellation and stale-refresh overwrite assertions. After the minimal
fix, the same command passed 2 files / 22 tests.

The release screenshot harness initially exposed two local verification issues:

1. Playwright CLI's `run-code` sandbox did not provide the global `URL`
   constructor. The ignored temporary script switched to deterministic prefix
   slicing for its fixed API URL.
2. Starting Vite through `npm.cmd` left the child `node vite.js` listener after
   the parent stopped. Cleanup identified the exact workspace/port command line,
   stopped only that child, and verified zero listeners.

These were temporary harness issues, not application defects. The ignored
script, Playwright session data, Vite logs, and listeners were removed. No
temporary debugging artifact is part of the release candidate.
