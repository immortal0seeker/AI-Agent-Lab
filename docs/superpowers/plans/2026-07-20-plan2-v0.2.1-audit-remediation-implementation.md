> **For agentic workers:** REQUIRED SUB-SKILL: Use `executing-plans` to implement this plan inline. Subagents are forbidden by the active project constraint. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix every post-release Plan 2 audit finding that belongs inside the Tool Calling + Simple Agent Loop boundary and prepare a verified `v0.2.1` patch without moving the published `v0.2.0` tag.

**Architecture:** Harden the existing Tool boundary before persistence, make the Simple Agent budget and timeout explicit, add strict ToolCall ordering through one backward-compatible migration, and recover completed synchronous runs through tab-scoped frontend state. Preserve the current thin routes, SQLite-first architecture, Provider abstraction, read-only Tool set, and deferred later-Plan capabilities.

**Tech Stack:** Python 3.11, FastAPI, Pydantic 2, JSON Schema Draft 2020-12, SQLAlchemy 2, Alembic, SQLite, pytest 9, React 19, TypeScript 5.9, Vitest 4, jsdom, Vite 7.

```python
class StandardJsonError(ValueError):
    pass


def ensure_standard_json(value: object, *, max_bytes: int | None = None) -> None:
    try:
        encoded = json.dumps(
            value,
            ensure_ascii=False,
            separators=(",", ":"),
            allow_nan=False,
        ).encode("utf-8")
    except (TypeError, ValueError) as exc:
        raise StandardJsonError("value must contain only standard JSON") from exc
    if max_bytes is not None and len(encoded) > max_bytes:
        raise StandardJsonError("value exceeds the allowed JSON size")
```
